# 03 — Manarah OS

## التعريف

Manarah OS هو النواة التقنية المشتركة التي تعمل فوقها منتجات منارة. الهدف منه منع تكرار الوظائف الأساسية داخل كل منتج، وتحويل منارة إلى منصة قابلة للتوسع.

## لماذا نحتاج Manarah OS؟

بدون نواة مشتركة، سيصبح كل منتج مستقلًا:

- حسابات مكررة.
- صلاحيات مكررة.
- تقارير مكررة.
- تكاملات مكررة.
- بيانات غير مترابطة.

Manarah OS يحل ذلك بتوفير طبقة مشتركة لكل المنتجات.

## الخدمات الأساسية

### 1. Identity

- تسجيل دخول.
- ملف مستخدم.
- ربط المستخدم بعدة مؤسسات.
- التحقق من البريد.
- إدارة الجلسات.

### 2. Organizations

- مؤسسة.
- فروع.
- فرق.
- أعضاء.
- إعدادات.
- خطط اشتراك لاحقًا.

### 3. Roles & Permissions

- أدوار جاهزة.
- صلاحيات مخصصة.
- ربط المستخدم بدور داخل مؤسسة.
- صلاحيات على مستوى الموارد.

### 4. Forms Engine

- إنشاء نماذج.
- حقول مخصصة.
- تحقق من المدخلات.
- ربط النماذج ببرنامج أو مستفيد أو تقرير.

### 5. Workflow Engine

- خطوات عمل.
- موافقات.
- حالات.
- تحويلات بين الفرق.
- إشعارات تلقائية.

### 6. Documents

- رفع ملفات.
- تصنيف.
- ربط بمؤسسة/برنامج/مستفيد.
- صلاحيات عرض.
- أرشفة.

### 7. Reports & Dashboards

- مؤشرات أساسية.
- تقارير دورية.
- تصدير PDF/CSV لاحقًا.
- قوالب تقارير.

### 8. Impact Layer

- تعريف مؤشرات.
- ربط المؤشرات بالبرامج.
- تتبع القيم عبر الزمن.
- حساب النتائج.

### 9. Notifications

- إشعارات داخلية.
- بريد إلكتروني لاحقًا.
- تنبيهات مهام.
- تذكيرات موافقات.

### 10. Audit Log

- من فعل ماذا؟
- متى؟
- في أي مؤسسة؟
- على أي مورد؟

### 11. Integrations

- REST API.
- Webhooks.
- مفاتيح API.
- تكامل مع أنظمة خارجية لاحقًا.

## مبادئ التصميم

- Multi-tenant من البداية.
- الصلاحيات أولًا.
- كل عملية مهمة لها Audit Log.
- كل كيان قابل للربط بالأثر.
- لا ذكاء اصطناعي بدون بيانات منظمة.

## نسخة Core المطلوبة للـ MVP

لأول إصدار نحتاج فقط:

- Users.
- Organizations.
- Organization Members.
- Roles.
- Basic Permissions.
- Programs.
- Beneficiaries.
- Volunteers.
- Documents.
- Impact Metrics.
- Basic Dashboard.
- Audit Log مبسط.
