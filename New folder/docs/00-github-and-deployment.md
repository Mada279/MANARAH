# 00 — خطوات GitHub والنشر

## الوضع الحالي

تم إنشاء Pull Request سابقًا لتعديلات الموقع والـ README:

- الفرع: `arena/019f495e-manarah`
- العنوان: `Align MANARAH website with ecosystem vision`

> ملاحظة: هذه الجلسة قد تكون مقيدة من ناحية عمليات GitHub البعيدة بعد إغلاقها أو دمجها. إذا لم تنجح عمليات push/merge/gh من داخل الجلسة، يتم تنفيذ الخطوات التالية يدويًا من GitHub أو في جلسة Arena جديدة.

## الخطوة 1: مراجعة Pull Request

راجع الآتي قبل الدمج:

- عنوان الصفحة والرسالة الأولى.
- الأقسام الجديدة: الرؤية، المنظومة، Manarah OS، الشركاء.
- عدم وجود تركيز زائد على Blockchain/DeFi.
- نجاح البناء `npm run build`.

## الخطوة 2: الدمج إلى main

من GitHub:

1. افتح Pull Request.
2. راجع الملفات المعدلة.
3. اضغط **Merge pull request**.
4. احذف الفرع فقط إذا لم تعد الجلسة تحتاجه.

## الخطوة 3: تفعيل GitHub Pages

من إعدادات المستودع:

1. Settings.
2. Pages.
3. اختر مصدر النشر المناسب.
4. إن كان المشروع يستخدم `gh-pages` فالأمر المحلي هو:

```bash
npm run deploy
```

## الخطوة 4: اختبار الموقع المنشور

افتح الرابط المتوقع:

```txt
https://mada279.github.io/MANARAH/
```

وتحقق من:

- تحميل الصفحة بدون شاشة بيضاء.
- ظهور صور الخلفية.
- عمل التنقل الداخلي.
- توافق الموبايل.
- عدم وجود نصوص قديمة مثل ManaraPay أو ZakatChain أو DeFi كرسالة أساسية.

## الخطوة 5: لاحقًا — نشر تلقائي

الأفضل لاحقًا إضافة GitHub Actions للنشر التلقائي عند كل تحديث على `main`، بدل الاعتماد على النشر اليدوي.
