# 08 — قائمة مراجعة الإطلاق

## قبل بناء المنتج

- [ ] دمج PR الخاص بالموقع.
- [ ] نشر الموقع التعريفي.
- [ ] مراجعة الرسالة البصرية والنصية.
- [ ] اعتماد Mishkat MVP كأول منتج.
- [ ] اختيار مؤسسة تجريبية أو سيناريو واقعي.

## قبل تنفيذ Backend

- [ ] اعتماد نموذج البيانات الأولي.
- [ ] اختيار PostgreSQL.
- [ ] اختيار ORM.
- [ ] تحديد طريقة المصادقة.
- [ ] تحديد الصلاحيات الأولية.

## قبل تنفيذ Dashboard

- [ ] اعتماد الشاشات الأساسية.
- [ ] تحديد أول 5 مؤشرات أثر.
- [ ] تحديد الحقول الأساسية للمستفيد.
- [ ] تحديد الحقول الأساسية للمتطوع.
- [ ] تحديد حالات البرنامج.

## قبل تجربة مؤسسة حقيقية

- [ ] إنشاء بيانات تجريبية.
- [ ] اختبار صلاحيات المستخدمين.
- [ ] اختبار إنشاء برنامج.
- [ ] اختبار إضافة مستفيدين ومتطوعين.
- [ ] اختبار مؤشرات الأثر.
- [ ] اختبار التقرير الأولي.

## مؤشرات نجاح الإطلاق التجريبي

- [ ] مؤسسة واحدة تستخدم النظام أسبوعين على الأقل.
- [ ] إدخال برنامجين أو أكثر.
- [ ] إدخال 50 مستفيدًا أو أكثر.
- [ ] إدخال 10 متطوعين أو أكثر.
- [ ] متابعة 5 مؤشرات أثر على الأقل.
- [ ] استخراج تقرير واحد مفيد.

## أسئلة ما بعد التجربة

1. هل وفّر النظام وقتًا على المؤسسة؟
2. هل أصبح الأثر أوضح؟
3. هل واجهة الاستخدام مفهومة؟
4. ما أكثر شاشة استخدموها؟
5. ما أكثر شيء ناقص؟
6. هل سيدفعون أو يلتزمون باستخدام نسخة محسنة؟
