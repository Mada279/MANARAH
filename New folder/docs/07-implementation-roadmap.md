# 07 — خارطة التنفيذ

## الهدف

بناء نسخة أولى من Mishkat MVP قابلة للتجربة مع مؤسسة حقيقية خلال 4 إلى 6 أسابيع.

## افتراضات تقنية

### Frontend

- React.
- TypeScript.
- Tailwind CSS.
- Vite.

### Backend المقترح

- Node.js.
- Express أو NestJS.
- PostgreSQL.
- Prisma ORM.
- JWT/Auth.

### لاحقًا

- Queue للمهام.
- Object Storage للملفات.
- AI services.
- Webhooks.

## المرحلة A — تجهيز الأساس

المدة: 3 إلى 5 أيام.

المهام:

- تحديد بنية المشروع.
- إعداد Backend.
- إعداد قاعدة البيانات.
- إعداد Prisma.
- بناء Auth بسيط.
- بناء Organization model.

المخرجات:

- مستخدم يستطيع الدخول.
- مؤسسة يمكن إنشاؤها.
- ربط المستخدم بالمؤسسة.

## المرحلة B — البرامج والمؤسسة

المدة: أسبوع.

المهام:

- CRUD للبرامج.
- أعضاء المؤسسة.
- أدوار أولية.
- لوحة مؤسسة بسيطة.

المخرجات:

- إنشاء برنامج.
- عرض البرامج.
- إحصاءات أولية.

## المرحلة C — المستفيدون والمتطوعون

المدة: أسبوع.

المهام:

- CRUD للمستفيدين.
- CRUD للمتطوعين.
- ربطهم بالبرامج.
- تحديث ساعات التطوع.

المخرجات:

- جدول مستفيدين.
- جدول متطوعين.
- إحصاءات متصلة باللوحة.

## المرحلة D — مؤشرات الأثر

المدة: أسبوع.

المهام:

- تعريف مؤشرات.
- إدخال قيم.
- حساب تقدم مقابل الهدف.
- عرض بطاقات ورسوم بسيطة.

المخرجات:

- Dashboard أثر أولي.
- مؤشرات قابلة للتحديث.

## المرحلة E — التقارير والمستندات

المدة: أسبوع.

المهام:

- إنشاء تقرير أثر لفترة محددة.
- عرض ملخص البرنامج.
- رفع مستندات وربطها ببرنامج.
- تصدير بسيط لاحقًا.

المخرجات:

- تقرير برنامج.
- أرشيف مستندات.

## المرحلة F — تجربة مؤسسة حقيقية

المدة: أسبوع إلى أسبوعين.

المهام:

- اختيار مؤسسة تجريبية.
- إدخال بيانات حقيقية.
- مقابلة المستخدمين.
- تسجيل الملاحظات.
- ترتيب أولويات التحسين.

المخرجات:

- قائمة مشاكل واقعية.
- تحسينات UX.
- قرار واضح: هل نتابع أم نعيد التوجيه؟
