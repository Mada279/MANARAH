# 25 — تقرير الطالب

## الهدف

إضافة تقرير تفصيلي لكل طالب داخل إدارة المدارس ودور التحفيظ في مشكاة، بحيث يستطيع المشرف أو المعلم معرفة حالة الطالب بسرعة واتخاذ قرار متابعة مناسب.

## Endpoint جديد

```http
GET /api/v1/organizations/:organizationId/education/students/:studentId/report
```

## ماذا يعرض التقرير؟

- بيانات الطالب.
- الفصل أو الحلقة المرتبط بها.
- ولي الأمر إن وجد.
- التقدم العلمي.
- عدد صفحات الحفظ.
- نسبة الحضور.
- حالة المصروفات أو الكفالة.
- تنبيهات.
- توصيات.

## أمثلة على التنبيهات

- حضور أقل من 85%.
- تقدم علمي أقل من 70%.
- مستحقات دراسية غير مسددة.

## أمثلة على التوصيات

- متابعة ولي الأمر.
- خطة دعم علمي قصيرة.
- زيادة ورد الحفظ والمراجعة.
- توثيق أثر الكفالة التعليمية.

## تحديث Dashboard

داخل تبويب:

```txt
التعليم والتحفيظ
```

أصبح بجانب كل طالب زر:

```txt
عرض تقرير الطالب
```

عند الضغط عليه تظهر بطاقة تقرير الطالب أعلى قائمة الطلاب.

## تحديث Smoke Test

تم تحديث الاختبار ليتحقق من:

```http
GET /education/students/:studentId/report
```

بعد إنشاء طالب تجريبي، ويتأكد أن التقرير يحتوي على توصيات.

## المرحلة التالية

يمكن لاحقًا تطوير التقرير إلى:

- تقرير PDF.
- إرسال التقرير لولي الأمر.
- سجل متابعة شهري.
- خطة علاجية للطالب.
- ربط التقرير بالحضور والاختبارات والحفظ والمراجعة.
