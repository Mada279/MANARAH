# 22 — Audit Log وسجل التدقيق

## الهدف

إضافة سجل تدقيق أولي داخل Manarah OS / Mishkat MVP لتوثيق العمليات المهمة ومعرفة من قام بماذا ومتى.

## ما تم تنفيذه

تمت إضافة نوع جديد في JSON Store:

```ts
AuditLog
```

ويحفظ:

- المؤسسة المرتبطة بالعملية.
- المستخدم المنفذ.
- البريد الإلكتروني للمستخدم.
- نوع العملية.
- نوع الكيان.
- معرف الكيان.
- بيانات إضافية `metadata`.
- تاريخ التنفيذ.

## العمليات التي يتم تسجيلها الآن

- تسجيل الدخول.
- إعادة تهيئة المتجر المحلي.
- إنشاء مؤسسة.
- إنشاء / تعديل / حذف برنامج.
- إنشاء / تعديل / حذف مستفيد.
- إنشاء / تعديل / حذف متطوع.
- إنشاء / تعديل / حذف مؤشر أثر.

## Endpoint جديد

```http
GET /api/v1/organizations/:organizationId/audit-logs?limit=50
```

يرجع آخر سجلات التدقيق الخاصة بالمؤسسة.

## تحديث الواجهة

تم تحديث لوحة مشكاة المستقلة لعرض آخر سجلات التدقيق في تبويب:

```txt
نظرة عامة
```

كما يتم عرض عدد السجلات في تبويب:

```txt
الإعدادات
```

## تحديث Smoke Test

تم تحديث:

```txt
scripts/smoke-test.mjs
```

ليتحقق من وجود سجل:

```txt
program.create
```

بعد إنشاء برنامج تجريبي.

## ملاحظات مهمة

هذا Audit Log أولي ومناسب للـ MVP المحلي. في مرحلة PostgreSQL يجب نقله إلى جدول:

```txt
audit_logs
```

مع تحسينات:

- تسجيل IP / User Agent.
- تسجيل before/after للعمليات الحساسة.
- منع حذف سجلات التدقيق.
- تصفية وبحث وتصدير.
- ربط السجلات بالصلاحيات.
