# 15 — الاختبار المحلي وSmoke Test

## الهدف

إضافة طريقة سريعة للتأكد من أن API تعمل بعد كل تعديل.

## السكربت الجديد

```bash
npm run api:smoke
```

## طريقة التشغيل

افتح نافذة أولى:

```bash
npm run server:dev
```

ثم افتح نافذة ثانية:

```bash
npm run api:smoke
```

## ماذا يختبر؟

الاختبار يقوم بالآتي:

1. فحص `/health`.
2. تسجيل دخول تجريبي عبر `/api/v1/auth/login`.
3. جلب المؤسسات.
4. إنشاء برنامج تجريبي.
5. إنشاء مستفيد تجريبي.
6. إنشاء متطوع تجريبي.
7. جلب Dashboard.
8. حذف البيانات التجريبية التي أنشأها الاختبار.

## متغير API

يمكن تغيير رابط API:

```bash
API_BASE_URL=http://localhost:4000 npm run api:smoke
```

## مكان الملف

```txt
scripts/smoke-test.mjs
```

## لماذا هذا مهم؟

قبل نقل المشروع إلى GitHub أو قبل أي تطوير جديد، يمكن تشغيل Smoke Test للتأكد أن المسارات الأساسية لم تنكسر.
