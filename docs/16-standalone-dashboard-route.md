# 16 — لوحة مشكاة المستقلة

## الهدف

فصل تجربة Mishkat MVP عن الصفحة التسويقية، بحيث تصبح لوحة التحكم مساحة مستقلة يمكن تطويرها لاحقًا كتطبيق Dashboard كامل.

## ما تم تنفيذه

تمت إضافة مكوّن مستقل:

```txt
src/components/dashboard/DashboardApp.tsx
```

وتم تحديث:

```txt
src/App.tsx
src/components/Navbar.tsx
```

## طريقة الفتح

من الموقع اضغط:

```txt
لوحة مشكاة
```

أو افتح الرابط:

```txt
#dashboard
```

مثال محلي:

```txt
http://localhost:5173/MANARAH/#dashboard
```

## محتوى لوحة التحكم

تحتوي اللوحة على تبويبات مستقلة:

- نظرة عامة.
- البرامج.
- المستفيدون.
- المتطوعون.
- مؤشرات الأثر.
- الإعدادات.

## ما تدعمه اللوحة الآن

### البرامج

- عرض البرامج.
- إضافة برنامج.
- زيادة نسبة التقدم.
- حذف برنامج.

### المستفيدون

- عرض المستفيدين.
- إضافة مستفيد.
- تبديل حالة المتابعة.
- حذف مستفيد.

### المتطوعون

- عرض المتطوعين.
- إضافة متطوع.
- زيادة ساعات التطوع.
- حذف متطوع.

### مؤشرات الأثر

- عرض المؤشرات.
- إضافة مؤشر.
- زيادة القيمة الحالية.
- حذف مؤشر.

### الإعدادات

- عرض معلومات المؤسسة.
- تحديث البيانات.
- إعادة تهيئة بيانات التطوير.
- العودة إلى الموقع التعريفي.

## ملاحظات تقنية

- التوجيه الحالي يتم عبر hash route بسيط: `#dashboard`.
- لا نستخدم React Router بعد لتقليل التعقيد.
- يمكن لاحقًا تحويل اللوحة إلى route حقيقي مثل `/dashboard` عند ضبط الاستضافة.

## الخطوة التالية

بعد فصل لوحة التحكم، تصبح الخطوات القادمة:

1. تحسين تصميم الجداول والنماذج.
2. إضافة صفحة تسجيل دخول فعلية.
3. ربط صلاحيات المستخدم بالتبويبات والإجراءات.
4. استبدال JSON persistence بـ PostgreSQL.
5. فصل المشروع مستقبلًا إلى:

```txt
apps/marketing-site
apps/dashboard
packages/ui
packages/api-client
```
