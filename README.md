# منارة | MANARAH

> **منظومة رقمية لبناء الإنسان، وتمكين المؤسسة، وقياس الأثر، وضمان الاستدامة.**

منارة ليست تطبيقًا منفردًا، وليست منصة محتوى فقط، وليست نظام ERP تقليديًا. منارة هي رؤية لبنية رقمية متكاملة تربط الإنسان والمؤسسة والمعرفة والعمل المجتمعي والوقف والذكاء الاصطناعي داخل منظومة واحدة قابلة للنمو.

## نبذة مختصرة

**منارة** مشروع يهدف إلى بناء أكبر بنية رقمية تخدم الإنسان والمؤسسات ذات الرسالة، من خلال ربط التعليم، وإدارة المؤسسات، والوقف، والمعرفة، وقياس الأثر، والذكاء الاصطناعي داخل منصة واحدة.

تركّز منارة على أربعة محاور رئيسية:

1. **بناء الإنسان** عبر رحلة شخصية متكاملة.
2. **تمكين المؤسسة** عبر أدوات تشغيل وحوكمة وقياس أثر.
3. **تنظيم المعرفة** وتحويلها إلى مورد حي قابل للبحث والتعلّم.
4. **ضمان الاستدامة** عبر الوقف الرقمي ونماذج تشغيل مستدامة.

---

## الرؤية والرسالة

### الرؤية

أن تصبح منارة البنية الرقمية المرجعية عالميًا لتمكين الإنسان والمؤسسات ذات الرسالة، من خلال منظومة تقنية تجمع بين المعرفة، والتعليم، والعمل المجتمعي، والحوكمة، والاستدامة الوقفية، والذكاء الاصطناعي.

### الرسالة

تطوير منظومة رقمية متكاملة تساعد الفرد على النمو، وتمكّن المؤسسات من أداء رسالتها بكفاءة، وتربط المتبرعين والمستفيدين والمتطوعين في شبكة واحدة قائمة على الشفافية وقياس الأثر والاستدامة.

---

## القيم الحاكمة

- الإنسان أولًا.
- الإحسان والإتقان.
- الأمانة والشفافية.
- العدالة والرحمة.
- المعرفة والتمكين.
- الاستدامة والوقف.
- الابتكار المسؤول.
- التقنية في خدمة الرسالة، لا العكس.

## فلسفة البناء

منارة لا تُبنى حول تطبيق واحد، بل حول منظومة مترابطة:

```txt
                    الإنسان
                       │
المعرفة ───────── المجتمع ───────── المؤسسة
                       │
                   الاستدامة
```

أي قرار داخل المنظومة يجب أن يخدم واحدًا أو أكثر من هذه المحاور.

## Manarah OS

**Manarah OS** هو الأساس المشترك للمنظومة، وليس منتجًا ظاهرًا للمستخدم فقط. وظيفته توفير القدرات المركزية التي تعتمد عليها كل المنتجات.

### القدرات الأساسية

- الهوية الرقمية وإدارة الحسابات.
- المؤسسات والفروع والفرق.
- الأدوار والصلاحيات.
- النماذج والحقول المخصصة.
- سير العمل والموافقات.
- المستندات والأرشفة.
- الإشعارات والتنبيهات.
- التقارير ولوحات المؤشرات.
- قياس الأثر ومؤشرات الأداء.
- التكاملات وواجهات API.
- سجل التدقيق Audit Log.
- خدمات الذكاء الاصطناعي.
- طبقة المعرفة والبحث.
- تعدد المؤسسات Multi-tenancy.

الهدف من Manarah OS هو منع بناء منتجات منفصلة ومكررة، وتحويل منارة إلى منصة قابلة للتوسع لعشرات المنتجات خلال سنوات.

## المنتجات الأساسية

### 1. معراج | Meraj

منصة بناء الإنسان ورحلته الشخصية.

تشمل:

- الملف الشخصي الذكي.
- الرحلة الإيمانية.
- الرحلة العلمية.
- الرحلة المهنية والمهارية.
- العادات والأهداف والإنتاجية.
- التطوع والعطاء.
- المجتمع والمجموعات.
- مساعد ذكاء اصطناعي شخصي.
- سجل الإنجاز والأثر.

### 2. مشكاة | Mishkat

منصة تشغيل المؤسسات ذات الرسالة مثل الجمعيات، المدارس، الأوقاف، المساجد، المراكز، الأكاديميات، والمبادرات.

تشمل:

- إدارة المؤسسات والفروع.
- إدارة الموظفين والمتطوعين.
- إدارة البرامج والمشاريع.
- إدارة المستفيدين.
- إدارة التعليم والحلقات والدورات.
- إدارة التبرعات والشركاء.
- إدارة الوثائق والعقود.
- لوحات مؤشرات KPI.
- تقارير أثر.
- حوكمة وصلاحيات.

### 3. محرك الأثر | Impact Intelligence Engine

طبقة ذكاء لا تهدف فقط إلى عرض البيانات، بل إلى تحويلها إلى أثر قابل للقياس.

يقيس مثلًا:

- عدد المستفيدين.
- عدد الطلاب والمتعلمين.
- عدد المتطوعين وساعات التطوع.
- عدد البرامج والمبادرات.
- الأثر الاجتماعي والاقتصادي.
- كفاءة الإنفاق.
- قصص النجاح.
- العائد الاجتماعي على الاستثمار SROI.

### 4. الوقف الرقمي | Digital Waqf

منصة لإدارة الاستدامة طويلة الأجل، من خلال أصول وقفية وصناديق وبرامج تمويل شفافة.

تشمل:

- الأصول الوقفية.
- المصارف والمستفيدين.
- العوائد والتوزيع.
- الحوكمة والموافقات.
- التقارير المالية والأثرية.
- الربط مع مشكاة ومعراج ومحرك الأثر.

### 5. منصة المعرفة والأكاديمية

منصة لإدارة المحتوى والمعرفة والتعليم.

تشمل:

- مكتبة رقمية.
- دورات ومسارات تعلم.
- شهادات واختبارات.
- مقالات وأبحاث.
- فهرسة وبحث دلالي.
- ربط المعرفة بالمستخدم والمؤسسة والأثر.

### 6. بوابة المطورين والتكاملات

منارة لا تُبنى كنظام مغلق، بل كمنصة قابلة للتكامل.

تشمل:

- REST/GraphQL APIs.
- Webhooks.
- SDKs.
- توثيق تقني.
- إدارة مفاتيح API.
- بيئة Sandbox.

- المستثمرون.

## تشغيل المشروع محليًا

### المتطلبات

- Node.js
- npm

### التثبيت

```bash
npm install
```

أو لضمان تثبيت مطابق لملف القفل:

```bash
npm ci
```

### تشغيل واجهة الموقع

```bash
npm run dev
```

### تشغيل API التجريبية

```bash
npm run server:dev
```

لتعمل لوحة Mishkat Dashboard متصلة بالـ API، شغّل الأمرين في نافذتين منفصلتين:

```bash
npm run server:dev
npm run dev
```

ثم افتح صفحة الدخول من:

```txt
http://localhost:5173/MANARAH/#login
```

وسجّل بالمستخدم التجريبي:

```txt
admin@manarah.local
```

بعد الدخول سيتم تحويلك إلى:

```txt
http://localhost:5173/MANARAH/#dashboard
```

أو اضغط زر `لوحة مشكاة` من الموقع.

ويمكن ضبط رابط API عبر:

```env
VITE_API_URL="http://localhost:4000"
```

### فحص TypeScript للـ API

```bash
npm run server:check
```

### اختبار API محليًا

بعد تشغيل `npm run server:dev` في نافذة مستقلة:

```bash
npm run api:smoke
```

يتحقق الاختبار من أن المسارات المحمية ترفض الطلبات بدون:

```http
Authorization: Bearer demo-token
```

ثم يتحقق من أن مستخدم `viewer@manarah.local` يستطيع القراءة ولا يستطيع الكتابة، وبعدها يسجل دخول المدير ويرسل token مع طلبات CRUD.

### تشغيل PostgreSQL لاحقًا عبر Docker

```bash
docker compose up -d postgres
npm run db:generate
npm run db:migrate
npm run db:seed
```

### بناء نسخة الإنتاج

```bash
npm run build
```

### معاينة نسخة الإنتاج

```bash
npm run preview
```

### النشر عبر GitHub Pages

```bash
npm run deploy
```

## ملاحظات مهمة

1. **منارة ليست Blockchain أولًا**  
   يمكن استخدام البلوكتشين لاحقًا للشفافية والحوكمة والتوثيق، لكنه ليس هوية المشروع الأساسية.

2. **الأولوية للمنتج القابل للاستخدام**  
   البداية المقترحة هي Manarah OS Core ثم Mishkat MVP، لأن المؤسسات تحتاج أدوات تشغيل وقياس أثر قابلة للاستخدام مباشرة.

3. **الذكاء الاصطناعي طبقة مساعدة**  
   الذكاء الاصطناعي في منارة يجب أن يكون موجّهًا بالحوكمة والمعرفة الموثوقة، وليس بديلًا عن الإنسان أو المؤسسات.

4. **الوقف جزء من الاستدامة**  
   الوقف الرقمي ليس حملة تبرع، بل طبقة استدامة وحوكمة وتمويل طويل الأجل.

5. **قياس الأثر معيار النجاح**  
   نجاح منارة لا يُقاس بعدد الصفحات أو التطبيقات فقط، بل بقدرتها على بناء الإنسان، وتمكين المؤسسة، وقياس الأثر الحقيقي.

## الخلاصة

منارة مشروع لبناء منظومة رقمية حضارية طويلة المدى. تبدأ بمنصة تشغيل واضحة، ثم تتوسع تدريجيًا إلى الإنسان، والأثر، والوقف، والمعرفة، والمجتمع، والتكاملات.

```txt
Vision
  ↓
Manarah OS
  ↓
Mishkat MVP
  ↓
Impact Engine
  ↓
Meraj
  ↓
Digital Waqf
  ↓
Knowledge + Academy + Community
  ↓
Developer Platform + Ecosystem Integrations
```

**منارة: نبني الإنسان، ونمكّن المؤسسة، ونقيس الأثر، ونضمن الاستدامة.**
